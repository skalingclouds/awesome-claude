## Overview

**Phronesis** is a decision-assurance layer for AI agents, served as the `phronesis-hermes` MCP
server (streamable-HTTP, with an `mcp-remote` stdio bridge for Claude Desktop). An agent calls
Phronesis **before a high-stakes action** and gets back a calibrated, independently-benchmarked
**Decision Asset**, an auditable **action boundary** (`approve / review / block / seek-evidence`),
and a verifiable **Decision Receipt**. Phronesis is advisory/assurance: it never autonomously
transacts, binds, moves money, or modifies identity — the calling agent and its principal retain
the action.

## Connecting

All MCP tools require a Bearer JWT (free diligence tier — get one at
`https://api.phronesisintel.com/api/v1/signup`). Endpoint: `https://api.phronesisintel.com/mcp`.

```bash
claude mcp add --transport http phronesis-hermes https://api.phronesisintel.com/mcp \
  --header "Authorization: Bearer $PHRONESIS_JWT"
```

Claude Desktop (stdio client → remote, via the `mcp-remote` bridge):

```json
{
  "mcpServers": {
    "phronesis-hermes": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://api.phronesisintel.com/mcp", "--header", "Authorization: Bearer ${PHRONESIS_JWT}"],
      "env": { "PHRONESIS_JWT": "paste-your-JWT-from-/api/v1/signup" }
    }
  }
}
```

## Public read-only tools

- **`health_check`** — liveness + version probe.
- **`bench_query`** — Decision Quality Bench snapshot (eight modules; honest `insufficient_sample` where unscored).

Forecast, memory, and Decision-Asset mint tools require an entitled principal and are
deny-by-default gated. No-signup public data (read-only, REST not MCP):
`curl https://api.phronesisintel.com/.well-known/bench.json`.

## Source verification notes

Verified live on 2026-06-20:

- `https://api.phronesisintel.com/.well-known/mcp.json` (HTTP 200) advertises the
  `phronesis-hermes` server, the streamable-HTTP transport, and the public read-only tools
  `health_check` + `bench_query`.
- `POST https://api.phronesisintel.com/mcp` without a JWT returns `401` (`auth_error`) —
  confirming all MCP tools require authentication (no anonymous access).
- `https://api.phronesisintel.com/.well-known/bench.json` and `/health/live` return HTTP 200
  (the unauthenticated public read-only substrate).
- Docs: `https://phronesisintel.com/for-agents` (HTTP 200).

Phronesis is a closed-source hosted product; there is no public source repository. The
verifiable integration surface is the public MCP manifest, the OpenAPI spec at
`https://api.phronesisintel.com/.well-known/openapi.json`, and the docs linked above.
